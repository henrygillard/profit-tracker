<!-- Polaris Web Component patterns from research -->
Layout:
- s-page: top-level page wrapper
- s-section: card-like section container, accepts heading prop
- s-stack: flex layout with gap
- s-text: typography, variant prop (heading-md, body-sm, etc.)
- s-badge tone="warning": inline status indicator
- s-banner tone="warning": block-level warning

Table pattern (Pattern 6 from research):
- s-table > s-table-header-row > s-table-header (listSlot="primary"|"labeled", format="numeric")
- s-table > s-table-body > s-table-row > s-table-cell
- No built-in sort events — sort is client-side React state

Date range presets (implement on frontend, send ISO to backend):
- "Last 7 days", "Last 30 days", "Last 90 days" + custom from/to inputs
- Always compute from/to as explicit ISO timestamps before passing to API
