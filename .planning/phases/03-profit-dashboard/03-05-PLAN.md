From server.js (current /admin handler — lines 68-115):
```javascript
app.get('/admin', async (req, res) => {
  const shop = req.query.shop;
  if (!shop) return res.status(400).send('Missing shop parameter');
  const session = await prisma.shopSession.findFirst({ where: { shop } });
  if (!session) return res.redirect(`/auth?shop=${encodeURIComponent(shop)}`);
  res.send(`<!DOCTYPE html>...inline HTML placeholder...`);
});
```

Replace res.send() with (from research Pattern 3):
```javascript
res.sendFile(path.join(__dirname, 'public', 'app', 'index.html'));
```

The session check (findFirst + redirect) stays intact — only the final response changes.

Express static middleware (already in server.js line 48):
```javascript
app.use(express.static(path.join(__dirname, 'public')));
```
This already serves public/app/assets/* correctly. The /app/ base path in Vite config ensures assets load from /app/assets/ which the static middleware handles.

Anti-pattern: Do NOT add a wildcard SPA catch-all route that intercepts /api/* or /webhooks/*.
