    **MarginAlertBanner.jsx:**

    Create `web/src/components/MarginAlertBanner.jsx`. Props: `{ dateRange, onAtRiskCount }`.

    Behavior:
    - On mount and when dateRange.from or dateRange.to changes: fetch `/api/alerts/margin?from=...&to=...`
    - After fetch: call `onAtRiskCount(data.atRiskCount)` so App.jsx can drive the nav badge
    - Reset `dismissed` state to false when dateRange changes (new range = fresh context)
    - If `!data || data.atRiskCount === 0`: return null (no banner)
    - Split atRiskSkus into `criticalSkus` (isCritical=true) and `warningSkus` (isCritical=false)
    - CRITICAL section (if criticalSkus.length > 0): renders with `className="pt-alert pt-alert-danger"`, NO dismiss button (ALERT-03 invariant). Label: "CRITICAL: X SKU(s) losing money on every sale" followed by comma-separated `productName (marginPct%)`
    - WARNING section (if warningSkus.length > 0 && !dismissed): renders with `className="pt-alert pt-alert-warning"`, HAS a dismiss button (`onClick={() => setDismissed(true)}`). Label: "X SKU(s) below {threshold}% margin threshold" followed by comma-separated list.
    - On fetch error: silently ignore (same pattern as CogsCoverage — alerts are non-critical UI)
    - Import apiFetch from `../api.js`; use React, useState, useEffect from 'react'

    Key implementation detail for CRITICAL always visible:
    Render CRITICAL and WARNING as separate JSX elements. `dismissed` state only affects the WARNING element — the CRITICAL element has no `dismissed` condition. Do NOT wrap both in a single container that can be dismissed.

    **SettingsScreen.jsx:**

    Create `web/src/components/SettingsScreen.jsx`. No props needed.

    Behavior:
    - On mount: GET /api/settings → set threshold state
    - Renders a pt-card with title "Settings"
    - Shows a labeled number input: label="Margin Alert Threshold (%)", min=0, max=100, step=0.5
    - Shows a Save button. On click: PUT /api/settings { marginAlertThreshold: threshold } → set saved=true
    - When input changes: reset saved=false (cleared "Saved" confirmation)
    - When saved=true: show inline "Saved" confirmation text
    - While threshold=null (loading): input shows value 20 as placeholder (same as research pattern)
    - On fetch error in save: catch and show an error message (e.g. "Save failed")

    Follow the pt-card/pt-card-header/pt-card-body/pt-card-title class structure from existing components.
